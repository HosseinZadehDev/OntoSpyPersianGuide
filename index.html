معرفی آنتواسپای

Ontospy یک ابزار خط فرمان برای بررسی، تحلیل و مستندسازی مدل‌های RDF و OWL است. این ابزار می‌تواند فایل‌های آنتولوژی را از مسیر محلی یا نشانی وب دریافت کند، ساختار آن‌ها مانند کلاس‌ها، ویژگی‌ها، فضای نام‌ها و نمونه‌ها را استخراج کند و سپس گزارش، مستندات HTML و خروجی‌های قابل مشاهده برای ارائه و تحلیل وب معنایی تولید کند.

منابع نمونه برای آزمون ابزار

از این منابع برای آزمون دستورهای scan، gendocs، ser و مستندسازی استفاده کنید.

01نصب پایتون 3.11

روش اول: دانلود مستقیم پایتون 3.11 از سایت رسمی.

https://www.python.org/downloads/release/python-3110/

روش دوم: نصب با winget در PowerShell یا CMD.

winget install -e --id Python.Python.3.11
هنگام نصب، گزینه Add Python to PATH را فعال کنید.
02بررسی نسخه نصب‌شده پایتون

نمایش نسخه‌های نصب‌شده پایتون:

py -0p
03ایجاد پوشه پروژه و محیط ایزوله

ایجاد پوشه پروژه و ورود به آن:

mkdir OntospyProject cd OntospyProject

ایجاد محیط ایزوله با پایتون 3.11:

py -3.11 -m venv .venv

فعال‌سازی محیط ایزوله در PowerShell یا CMD:

.venv\Scripts\activate
04به‌روزرسانی pip و نصب Ontospy

به‌روزرسانی pip با همین نسخه فعال‌شده پایتون:

python -m pip install --upgrade pip

نصب Ontospy:

pip install ontospy
05بررسی نصب و فهرست دستورها

نمایش راهنمای اصلی Ontospy:

ontospy --help
scanاسکن آنتولوژی و نمایش گزارش ساختار فایل
gendocsتولید مستندات HTML، Markdown و خروجی‌های قابل مشاهده
libمدیریت کتابخانه محلی RDF و آنتولوژی‌های ذخیره‌شده
shellاجرای محیط تعاملی برای بررسی آنتولوژی
serتبدیل یا سریال‌سازی فایل RDF به قالب‌های دیگر
utilsابزارهای کمکی مثل کشف آنتولوژی و آزمون JSON-LD
06دستور scan: اسکن و تحلیل آنتولوژی

اسکن آنتولوژی محلی:

ontospy scan filename

اسکن آنتولوژی آنلاین:

ontospy scan url
-f, --formatتعیین قالب ورودی مانند turtle
-i, --individualsاستخراج نمونه‌های تعریف‌شده در آنتولوژی
-x, --extraاستخراج داده‌های ضمنی با استنتاج ساده
-r, --rawنمایش RDF خام دریافت‌شده
-e, --endpointمعرفی آدرس به عنوان نقطه پایانی SPARQL
-v, --verboseنمایش جزئیات بیشتر در خروجی

نمونه‌های کاربردی

ontospy scan filename -i ontospy scan filename -x -i ontospy scan filename -r
07دستور gendocs: تولید مستندات و مصورسازی

ابتدا پوشه خروجی را بسازید:

mkdir docs

تولید مستندات از فایل محلی:

ontospy gendocs filename -o docs --type 2 --title "TSE Ontology"

نمایش نوع مستندات و قالب‌های موجود:

ontospy gendocs --showtypes ontospy gendocs --showthemes
-o, --outputpathمسیر ذخیره خروجی مستندات
--typeنوع مستندات مانند تک‌صفحه‌ای یا چندصفحه‌ای
--titleعنوان سفارشی مستندات
--themeانتخاب قالب ظاهری Bootstrap
--showtypesنمایش نوع‌های خروجی قابل استفاده
--showthemesنمایش قالب‌های قابل استفاده
--nobrowserجلوگیری از باز شدن خودکار مرورگر
-l, --libانتخاب آنتولوژی از کتابخانه محلی
-i, --individualsنمایش نمونه‌ها در مستندات
-x, --extraافزودن داده‌های ضمنی به خروجی
--preflabelانتخاب qname یا برچسب برای عنوان موجودیت‌ها
--preflangانتخاب زبان ترجیحی برای برچسب‌های چندزبانه

نمونه‌های کاربردی

ontospy gendocs filename -o docs --type 1 --title "Single Page Docs" ontospy gendocs filename -o docs --type 2 --theme simplex ontospy gendocs filename -o docs --type 2 --preflabel label --preflang fa ontospy gendocs filename -o docs --type 2 -i -x --nobrowser ontospy gendocs -l -o docs --type 2
08دستور lib: مدیریت کتابخانه محلی آنتولوژی‌ها

ذخیره فایل یا نشانی آنتولوژی در کتابخانه محلی:

ontospy lib --save filename ontospy lib --save url
--saveذخیره فایل محلی یا آنلاین در کتابخانه
--show, -sنمایش آنتولوژی‌های ذخیره‌شده
--bootstrapافزودن آنتولوژی‌های معروف به کتابخانه
--cacheپاک‌سازی حافظه پنهان کتابخانه
--directoryتغییر پوشه کتابخانه
--revealباز کردن پوشه کتابخانه در سیستم‌عامل
-i, --individualsنمایش نمونه‌ها هنگام بازبینی
-x, --extraنمایش داده‌های ضمنی هنگام بازبینی

نمونه‌های کاربردی

ontospy lib --show ontospy lib --show -i ontospy lib --bootstrap ontospy lib --cache ontospy lib --reveal ontospy lib --directory C:\OntospyLibrary
09دستور shell: محیط تعاملی Ontospy

اجرای shell بدون فایل:

ontospy shell
این دستور برای لینوکس است.

اجرای shell همراه با فایل آنتولوژی:

ontospy shell filename
کاربرد shell برای بررسی تعاملی کلاس‌ها، ویژگی‌ها، فضای نام‌ها و اطلاعات آنتولوژی است.
10دستور ser: تبدیل قالب RDF

تولید خروجی Turtle، XML، N-Triples یا JSON-LD:

ontospy ser filename -f turtle ontospy ser filename -f xml ontospy ser filename -f pretty-xml ontospy ser filename -f nt ontospy ser filename -f json-ld
-f, --output_formatتعیین قالب خروجی
-v, --verboseنمایش جزئیات اجرا
11دستور utils: ابزارهای کمکی

جست‌وجوی آنتولوژی در مخزن‌های آنلاین:

ontospy utils --discover

آزمون فایل JSON-LD:

ontospy utils --jsonld file.jsonld
--discover, -dپیدا کردن آنتولوژی آنلاین
--jsonld, -jبررسی فایل JSON-LD با محیط آزمون آنلاین
12پیاده‌سازی یک سناریوی کامل به عنوان نمونه
گام اول: نصب ابزارپایتون و آنتواسپای را نصب کنید.
گام دوم: اسکن آنتولوژیفایل یا لینک آنتولوژی را اسکن کنید.
گام سوم: مصورسازی و مستندسازیبا gendocs مستندات یا مصورسازی را تولید کنید.
گام چهارم: تغییر فرمتبا ser قالب فایل را تغییر دهید.

دستور نمونه برای اجرای پروژه

py -3.11 -m venv .venv .venv\Scripts\activate python -m pip install --upgrade pip pip install ontospy ontospy --help ontospy scan filename ontospy scan url mkdir docs ontospy gendocs filename -o docs --type 2 --title "Ontology Docs" ontospy gendocs url -o docs --type 2 --title "Online Ontology" ontospy ser filename -f json-ld
13خطاهای رایج و راه‌حل سریع
ontospy not recognized
محیط مجازی فعال نیست یا نصب کامل نشده است.
pip not found
از دستور python -m pip استفاده کنید.
Invalid format
قالب را از میان xml، n3، turtle، nt، pretty-xml و json-ld انتخاب کنید.
Output path error
پیش از اجرای gendocs، پوشه docs را با دستور mkdir docs ایجاد کنید.
